# AI Collector — основные команды проекта

## Установка зависимостей

```bash
npm install
```

Устанавливает зависимости из `package.json`.

Используется после клонирования проекта или после изменения зависимостей.

---

## Запуск проекта в dev-режиме

```bash
npm run dev
```

Запускает Vite и Electron-приложение в режиме разработки.

Использовать для ежедневной работы.

---

## Остановка dev-сервера

```txt
Ctrl + C
```

Останавливает запущенный процесс в терминале.

---

## Сборка проекта

```bash
npm run build
```

Запускает:

```txt
tsc
vite build
electron-builder
```

`electron-builder` создает desktop-сборку в `release/`.

Если Electron нужной версии еще не скачан, build может обратиться к GitHub. В Codex-песочнице для этого может понадобиться разрешение на сеть.

Во время сборки сейчас возможны предупреждения:

- Sass `legacy-js-api`;
- в `package.json` нет `description`;
- в `package.json` нет `author`;
- не задан app icon;
- macOS code signing может быть пропущен.

Эти предупреждения не равны ошибке сборки.

---

## Предпросмотр Vite-сборки

```bash
npm run preview
```

Запускает предпросмотр собранного Vite-приложения.

Для Electron-разработки используется редко, основная команда — `npm run dev`.

---

## Проверка кода ESLint

```bash
npm run lint
```

Фактически запускает:

```bash
eslint . --ext ts,tsx --report-unused-disable-directives --max-warnings 0
```

Проверяет код на ошибки и проблемные паттерны.

---

## Запуск автотестов

```bash
npm test
```

Единая команда для всех автотестов проекта.

Сейчас запускает UI-тесты.

---

## Запуск UI-тестов

```bash
npm run test:ui
```

Запускает автотесты UI-компонентов.

Текущая структура:

```txt
tests/ui/ComponentName/
  ComponentName.fixture.html
  ComponentName.fixture.tsx
  ComponentName.spec.mjs
```

---

## Проверка Button отдельно

```bash
npm run test:button
```

Запускает только тест общего компонента `Button`.

Проверяет:

- SVG-иконка внутри кнопки отрисовалась;
- клик по активной кнопке запускает функцию;
- клик по disabled-кнопке не запускает функцию;
- для icon-only кнопки есть временный tooltip.

---

## Установка Playwright-браузеров

```bash
npx playwright install
```

Скачивает браузеры для Playwright: Chromium, Firefox, WebKit и служебные зависимости.

Обычно выполняется один раз после установки Playwright.

---

## Форматирование файлов Prettier

В `package.json` пока нет отдельного `format` script.

Использовать:

```bash
npx prettier . --write
```

Форматирует файлы проекта по правилам из `package.json`.

---

## Проверка форматирования без изменения файлов

```bash
npx prettier . --check
```

Проверяет, какие файлы не соответствуют правилам форматирования, но не изменяет их.

---

## Проверка версии Prettier

```bash
npx prettier --version
```

Помогает проверить, что npm-пакет Prettier установлен в проекте.

---

## Проверка установленных пакетов

```bash
npm list --depth=0
```

Показывает установленные зависимости верхнего уровня.

---

## Проверка устаревших пакетов

```bash
npm outdated
```

Показывает пакеты, для которых есть новые версии.

Не обновлять автоматически без понимания совместимости Electron/Vite.

---

## Проверка уязвимостей

```bash
npm audit
```

Показывает предупреждения безопасности по зависимостям.

Не запускать:

```bash
npm audit fix --force
```

Эта команда может обновить зависимости до несовместимых версий и сломать Electron/Vite-шаблон.

---

## Очистка node_modules при проблемах

```bash
rm -rf node_modules package-lock.json
npm install
```

Использовать только если зависимости явно сломались и пользователь согласен на такую очистку.

После этого снова проверить:

```bash
npm run dev
```

---

## SCSS

SCSS работает через пакет:

```bash
npm install -D sass-embedded
```

Повторно запускать эту команду не нужно, если пакет уже есть в `package.json`.

Глобальный входной файл стилей:

```txt
src/app/styles/index.scss
```

Он подключается в:

```txt
src/main.tsx
```

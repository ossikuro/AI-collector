# src/core/provider-registry

Provider Registry — модуль, который отвечает за список доступных AI-сервисов.
Он скрывает от остального приложения, где и как хранятся описания провайдеров.
UI и другие модули не должны напрямую искать файлы `provider.json` внутри `src/providers`.
Вместо этого они должны обращаться к функциям-командам Provider Registry.

## Что делает модуль

- возвращает список всех провайдеров;
- возвращает одного провайдера по id;
- проверяет, существует ли провайдер;
- фильтрует провайдеров по категории.

## Что модуль не делает

- не хранит аккаунты пользователя;
- не открывает сайты;
- не запускает Playwright;
- не читает кредиты;
- не выполняет автоматизацию;
- не содержит React-компоненты.

## Почему это отдельный модуль

Сейчас провайдеры могут храниться в JSON-файлах внутри `src/providers`.

Позже способ хранения может измениться: например, провайдеры могут загружаться из локального файла, базы данных или пользовательского каталога.

Остальное приложение не должно зависеть от этого решения.
